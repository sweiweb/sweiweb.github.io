托管hexo博客

创建 github 仓库

首先登入 github，在右上角+点击 New repository 创建一个仓库，在 repository name 中输入 github账户名.github.io ，同时勾选下方的Public (否则无法开启github pages)和 Initialize this repository with a README

github龟速？可同时同步至国内托管平台 gitee:https://gitee.com/

创建 gitee 仓库

首先登入 gitee，在右上角+点击 新建仓库 ，在 仓库名称 栏中输入 gitee账户名 ，勾选下方的使用readme文件初始化这个仓库 ，点击创建

  • 开启gitee pages

进入仓库，在仓库的服务 -> gitee pages 中点击创建即可

连接 hexo 和 git

在终端输入命令设置与 github 一致的用户名称和邮箱地址

使用 ssh-keygen 生成私钥和公钥

注意查看输出信息的共钥id_rsa.pub所在地址,可以用命令cat ~/生成地址/id_rsa.pub查看共钥内容

  • github

登录 github，点击右上角头像下的settings，点击 SSH and GPG keys 菜单下的 New SSH key 添加 ssh，将 id_rsa.pub 文件里的内容复制上去。随后，可以在终端输入命令 ssh -T git@github.com 测试添加 ssh 是否成功。

  • gitee

登录 gitee，点击右上角头像下的设置，点击 SSH 公钥 菜单， 将 id_rsa.pub 文件里的内容复制到 公钥 下方，点击确定。随后，可以在终端输入命令 ssh -T git@gitee.com 测试添加 ssh 是否成功

配置Deployment

在 blog 目录下对文件 _config.yml 进行编辑，输入 vi _config.yml打开文件，输入i进入编写状态，修改文件末尾的 Deployment

其中的 repository 可以在 github (gitee) 账户下 your repositories (仓库) 的 clone or downlord (克隆/下载) 中，选择 Ues SSH (ssh) 查看。修改完成按ESC后输入 :wq 保存并退出（以下文件编辑步骤完全相同）

语法

markdown 语法

hexo 的文章内容采用 markdown 语法，撰写的文章内容应基于 markdown ，基本语法请在1分钟内学会

HTML 语法

有时候只靠 markdown 写出来的风格比较单调，比如改变字体格式和布局时，这时候就要用到 HTML 语法

HTML（HyperText Mark-up Language）是一种制作万维网页面的标准语言，它包括一系列标签，通过这些标签可以将网络上的文档格式统一，使分散的Internet资源连接为一个逻辑整体。HTML文本是由HTML命令组成的描述性文本，HTML命令可以说明文字，图形、动画、声音、表格、链接等

有关 HTML 的常用语法，参考本站 : HTML常用语法
